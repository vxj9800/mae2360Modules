Operators and Control Statements

Relational Operators

Relational operators are used in logical claims to evaluate conditions. These statements evaluate to either true/1 or false/0.

Syntax Result true if
a < b a is less than b
a <= b a is less than or equal to b
a > b a is greater than b
a >= b a is greater than or equal to b
a == b a is exactly equal to b
a != b (a ~= b) a is not equal to b (Matlab)

Logical Operators

Logical operators are used to evaluate multiple conditions.

Syntax Result true if
a && b a and b both are true
a || b a or b is true
!a (~a) a is not true

The ! operator can be combined with more than just =.
Example: !< means not less than.

Increment/Decrement/Assignment Operators

The C language has a long history of minimalism. Many of these commands can be accomplished by other means, but these commands are designed to make your code compact.
Syntax Meaning
a++, ++a Value of a is incremented by 1. Same as a = a + 1;
a--, --a Value of a is decremented by 1. Same as a = a - 1;
Syntax Meaning
b = a Value of a is assigned to b
b = ++a Value of a is incremented by 1 and then assigned to b
b = a++ Value of a is assigned to b and then incremented by 1
b = --a Value of a is decremented by 1 and then assigned to b
b = a-- Value of a is assigned to b and then decremented by 1
Syntax Meaning
b = a Value of a is assigned to b
b += a Value of b + a is assigned to b. Same as b = b + a;
b -= a Value of b - a is assigned to b. Same as b = b - a;
b *= a Value of b * a is assigned to b. Same as b = b * a;
b /= a Value of b / a is assigned to b. Same as b = b / a;
b %= a Value of b % a is assigned to b. Same as b = b % a;

Control statements

All of these tools are often used with control statements to form logical operations. The common control statements are:

  • if / else if
  • for
  • while
  • do while
  • switch

if / else if Statement

A block is a set of code contained within {...}. That block can be triggered when the conditions within an if statement are satisfied.

The else statement allows triggering another block of code when the conditions within an if statement are not satisfied.
